Do it Today: backend Endpoints

GET Endpoints

  • GET /
    Description: Returns an HTML page listing all the endpoints.



POST Endpoints

  • POST /createUser
    Description: Creates a new user.
    Parameters: 
    
{
    firstName: 'John',
    lastName: 'Doe',
    email: 'john@example.com',
    password: 'secretpassword'
}
    Returns: A message saying the user was created or an error message.



  • POST /login
    Description: Logs in a user to the application.
    Parameters: 
    
{
    email: 'john@example.com',
    password: 'secretpassword'
}
    Returns: A message saying the user was logged in or an error message.



PUT Endpoints

  • PUT /updateUser
    Description: Updates user credentials.
    Parameters: 
    
{
    currentEmail: 'john@example.com',
    newEmail: 'john.doe@example.com',
    currentPassword: 'secretpassword',
    newPassword: 'newsecretpassword'
}
    Returns: A message saying the user was updated or an error message.



DELETE Endpoints

  • DELETE /deleteUser
    Description: Deletes a user from the database.
    Parameters: 
    
{
    email: 'john@example.com',
    password: 'secretpassword'
}
    Returns: A message saying the user was deleted or an error message.
